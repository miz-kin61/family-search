# 👨‍👩‍👧 親子 Not-Self 条件付け分析ツール V2.0

## 📋 概要

両親のヒューマンデザインから、子供が幼少期に受けたNot-Self条件付けを推測する分析ツール。

## ✨ 主な機能

### ✅ 実装済み（v2.0）

1. **Parquet 4分割データ対応**
   - 1900-1934年: HD_Archive_1.parquet
   - 1935-1969年: HD_Archive_2.parquet
   - 1970-2004年: HD_Archive_3.parquet
   - 2005年以降: HD_Archive_4.parquet

2. **チャネル・センター自動判定**
   - ゲートデータからチャネルを判定
   - チャネルから定義センターを自動計算
   - 36チャネル完全対応

3. **日付選択時の最適データ参照**
   - 年に応じて適切なParquetファイルを自動選択
   - 時刻不明者は12:00のデータを使用
   - 時刻指定者は最も近い時刻のデータを取得

4. **複数家族パターン保存**
   - 分析結果を保存
   - 過去の分析を読み込み
   - 比較分析が可能

5. **AI推論機能（準備完了）**
   - Claude APIで深い洞察を生成
   - 個人の要素から個別メッセージ
   - インカネーションクロス分析

6. **PDF出力（わかりやすい表現）**
   - 専門用語を控えた表現
   - クライアント向けレポート
   - ダウンロード可能

7. **個別メッセージ生成**
   - タイプ、プロファイル、定義型から
   - インカネーションクロス分析
   - 重要チャネルの解説

## 📊 データ構造

### 入力データ（Parquet形式）

```
列名:
- JST_Time: 日本時間
- Type: タイプ（Generator, Projector等）
- Profile: プロファイル（1/3, 5/1等）
- Definition_Type: 定義型（Single, Split等）
- Authority: 権威（Sacral, Emotional等）
- Trigger: 変化トリガー
- P_Sun～P_Chiron: パーソナリティ（黒）の天体ゲート
- D_Sun～D_Chiron: デザイン（赤）の天体ゲート

形式: ゲート番号.ライン番号（例: 38.1）
```

### 出力データ

```python
{
    'gates': set(),                  # アクティブなゲート集合
    'active_channels': [],           # アクティブなチャネルリスト
    'defined_centers': [],           # 定義されたセンター
    'undefined_centers': [],         # 未定義センター
    'incarnation_cross': {},         # インカネーションクロス
    'type': str,                     # タイプ
    'profile': str,                  # プロファイル
    'definition': str,               # 定義型
    'authority': str                 # 権威
}
```

## 🎯 使い方

### 基本的な流れ

1. **タブ1: 新規分析**
   - 父・母・本人の生年月日を入力
   - 「分析を実行」ボタンをクリック
   - 「このパターンを保存」で保存可能

2. **タブ2: 保存パターン**
   - 過去の分析を表示
   - 「この分析を読み込む」で再分析

3. **タブ3: 分析結果**
   - 家族の基本情報
   - センター定義状態
   - Not-Self 条件付けの詳細

4. **タブ4: PDF出力**
   - わかりやすいレポート生成
   - ダウンロード可能

## 🐛 トラブルシューティング

### データが見つからない

**症状**: "HD_Archive_X.parquet が見つかりません"

**解決策**:
1. Parquetファイルが正しいディレクトリにあるか確認
2. ファイル名が正確か確認
3. サンプルモードで動作を確認

### チャネルが正しく判定されない

**症状**: センター定義が期待と異なる

**解決策**:
1. `hd_core_logic.py` の `CHANNELS` リストを確認
2. ゲートデータが正しいか確認
3. デバッグモードで中間データを確認

### PDF生成エラー

**症状**: PDFダウンロードが失敗

**解決策**:
1. reportlabがインストールされているか確認
2. 日本語フォントの設定を確認
3. エラーログを確認
